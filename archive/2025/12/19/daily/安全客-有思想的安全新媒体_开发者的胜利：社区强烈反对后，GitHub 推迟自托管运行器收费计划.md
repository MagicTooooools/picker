# 开发者的胜利：社区强烈反对后，GitHub 推迟自托管运行器收费计划

近日，代码托管平台 GitHub 发布博文宣布，自 2026 年 3 月 1 日起，其自动化部署工具 GitHub Actions 将额外收取平台费用。按照这一拟议调整，即便是开发者使用自托管运行器，也需按每分钟 0.002 美元的标准付费。该消息一经公布，随即引发了开发者社区的强烈抵制。

在 GitHub 口中所谓的 “热烈” 讨论之后，该公司宣布将推迟这项平台收费计划的实施。GitHub 在最新声明中承认，维持 Actions 控制层的运行确实产生实际成本，且公司一直持续为自托管运行器投入资源。但该公司也坦承，尽管这些现实情况不容忽视，此次推出收费计划的决定，并未充分征求开发者的反馈意见，最终未能达到预期目标。

目前，GitHub Actions 未来的定价调整细节尚未明确。GitHub 表示，需要更多时间与开发者沟通交流、收集反馈并重新评估收费方案，因此近期内不会对 GitHub Actions 收取平台费用。该公司称：“我们看到了大家的留言，也听取了大家的反馈。我们将推迟此前宣布的自托管 GitHub Actions 收费调整计划，以便有充足时间重新评估相关方案。”

此外，GitHub 还确认，自 2026 年 1 月 1 日起，将进一步下调官方托管运行器的费用，部分机型降价幅度最高可达 39%。尽管维持 Actions 控制层的运行始终存在成本，但该公司强调，会继续加大对自托管运行器的投入，确保其能在客户环境中规模化运行，尤其是适配复杂的企业应用场景。

展望未来，GitHub 表示将对 GitHub Actions 工具进行优化改进。一方面，会投入更多时间与开发者、客户及合作伙伴深入沟通，认真倾听各方观点；另一方面，会开启新的讨论渠道以收集更直接的反馈，这些反馈将用于规划 GitHub Actions 未来的发展路线。

GitHub 最后表示：“我们将持续推进 GitHub Actions 及整个平台的稳定运营，努力赢回大家的信任。”
