# 让 AI 写代码，它派了支咨询团队来

2025-12-31

> 这哪是实习生？这是咨询公司！

# 那个关于实习生的比喻

“AI就像一个能力很强但经验不足的实习生。”

这话你听过吧？我也听过，而且深以为然。

让AI帮我写段代码，它能嗖嗖嗖给你整出几百行——语法正确、逻辑通顺、注释齐全。但你总得盯着：这儿边界情况没考虑、那儿性能有问题、这个设计还得重构。跟带实习生似的，又当老师又当监工。

累。

所以当我发现**BMad**的时候，内心毫无波澜。又一个AI工具嘛，能有多神？

# 藏在菜单里的”50个高级方法”

然后我点开了分析师的菜单。

```
[WS] 工作流状态      [BP] 头脑风暴
[RS] 研究分析        [PB] 产品简介
[DP] 项目文档化      [PM] 派对模式
```

我心想，这不就是快捷指令吗？

但当我真正开始用的时候，才发现事情不对。

让我分析个技术方案，它不是直接给我答案，而是先问我：

> “这个系统的核心挑战是什么？用户场景有哪些？技术约束是什么？”

让我梳理文档结构，它不是直接列大纲，而是引导我：

> “这篇文章的目标读者是谁？他们已知什么？最需要解决什么问题？”

这哪是实习生？

这分明是个**咨询顾问**。

# BMad的底层逻辑：它为何与众不同

深入了解后，我发现BMad的设计藏着几个精妙的底层逻辑：

## 1. 意图驱动规划

传统的AI交互是**指令式**的：“帮我写个API文档。”

BMad是**对话式**的：它先问”你要解决什么问题？""目标读者是谁？""最想传达的核心观点是什么？”

它不急着给答案，它要先理解你的**意图**。

## 2. 模块化工作流系统

BMad不是一个大而全的Prompt，而是一套**模块化的工作流**。每个步骤独立、可组合、可扩展。这不是”一个Prompt”，而是**一套系统**。

## 3. Agent人格系统

BMad不是冷冰冰的工具，它给每个Agent赋予了**人格**：

| Agent | 名字 | 角色 |
| --- | --- | --- |
| Analyst | Mary | 商业分析师 |
| Architect | Alex | 架构师 |
| Dev | Amelia | 开发者 |
| PM | Nicole | 产品经理 |
| Barry | Quick-Flow | 快速流开发者 |

每个Agent有自己的**沟通风格**、**工作原则**。Mary会像寻宝一样挖掘洞见，Amelia会严格按故事文件执行，Barry会直接告诉你”别废话”。

这不叫”AI助手”，这叫**AI团队**。

## 4. 菜单驱动执行

不是扔给你一个万能Prompt让你自己摸索，而是**给你一张菜单**。你选，AI执行。清晰、可控、有边界。

## 5. 结构化创意

BMad最让我震撼的，是它内置的**50种头脑风暴方法**。以下是几个代表性的方法及出处：

| 方法 | 类别 | 说明 | 出处 |
| --- | --- | --- | --- |
| **六顶思考帽** | 结构化 | 从六个平行视角思考问题 | [知乎 - 六顶思考帽](https://zhuanlan.zhihu.com/p/373565956) |
| **SCAMPER** | 结构化 | 七个创新检视维度 | [MBA智库 - SCAMPER](https://doc.mbalib.com/tag/%E5%85%AD%E9%A1%B6%E6%80%9D%E8%80%83%E5%B8%BD) |
| **第一性原理** | 创意 | 剥离假设，回归本质 | [知乎 - 第一性原理](https://zhuanlan.zhihu.com/p/361668978) |
| **Five Whys** | 深度 | 丰田五问法，追根溯源 | [知乎 - 根本原因分析](https://zhuanlan.zhihu.com/p/409138950) |
| **Yes And Building** | 协作 | 即兴戏剧式创意叠加 | [Improvisational Thinking](https://www.improvopedia.org) |

想用”第一性原理”思考？选。想来一次”六顶思考帽”？选。

这不是50个快捷指令，这是**50种思维方式**。

## 6. 完整AI开发团队

BMad是一个**完整的AI开发方法论体系**：

```
分析阶段 → 规划阶段 → 解决方案 → 实施阶段
   ↓           ↓           ↓           ↓
Analyst    UX Designer   PRD        Dev
Research   Product Brief  Architecture  Code Review
Product Brief  Stories    Quick-Flow
```

还有**PM**、**SM**、**Tech Writer**、**Party Mode**（多Agent群聊）。

这是一张完整的**AI开发团队**阵容。

# 就像在咨询公司接受咨询

想象一下：你走进麦肯锡的办公室，说我想做个技术方案。

好的，咨询顾问说。先别急，我们聊聊——

> 你的业务场景是什么？
> 技术约束有哪些？
> 未来可能怎么演进？

每一个问题都直指本质。每一次对话都在帮你厘清思路。

BMad给我的就是这种感觉。

它不急着给答案。它像个经验老道的顾问，抛出一个个好问题，帮你把模糊的想法捏成清晰的骨架，再填上有血有肉的内容。

从”帮我写个方案”，变成”我们一起来构思这个方案”。

一字之差，云泥之别。

# AI协作的新范式

带实习生，你得把需求想清楚了再提。不然大眼瞪小眼，你急他也急。

找咨询公司，你只需要带着问题和困惑。剩下的，交给专业的人来引导。

BMad让我从”监工”变成了”导演”。

不再是一遍遍review代码，而是参与设计的过程。AI不是我的下属，而是我的搭档——一个比我更懂方法论、更有耐心的搭档。

50多个引导方法，不是50个代码生成器。

是50种帮你把问题想透彻的思维方式。

# 你的AI团队，值得更好的配置

说真的，用了BMad之后，我回不去了。

不是BMad多聪明，而是它够”懂行”。它知道好的方案怎么产出、好的架构怎么设计、好的技术决策怎么做。

它不是来帮你写代码的。

它是帮你把代码写对的。

所以下次有人说”AI就像实习生”，你可以回一句：

> “那是你的AI。我的AI，是一整个咨询团队。“
