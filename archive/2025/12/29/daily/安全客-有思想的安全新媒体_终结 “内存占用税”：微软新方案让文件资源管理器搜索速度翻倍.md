# 终结 “内存占用税”：微软新方案让文件资源管理器搜索速度翻倍

微软目前正在对 Windows 11 系统的文件资源管理器开展底层优化测试，目标是大幅降低其内存占用，并提升用户搜索图片、文档等文件时的响应速度。

当前，文件资源管理器的搜索功能往往会占用大量内存，且操作体验时常卡顿。该功能本质上依托于 Windows 系统的索引机制 —— 系统预先为文件建立索引，在用户执行搜索操作时再调取匹配结果。此次推出的底层优化，核心在于**消除冗余的索引操作**，以此提升搜索性能。具体来说，优化后的机制会避免对同一文件或文件夹的重复扫描，从而大幅减少系统的无效运算量。

这些优化措施已率先应用于 **Windows 11 Build 26220.7523 及更高版本**。微软指出，尽管现有的 Windows 搜索索引器已采用了特定的启发式算法，但在某些特定条件下，仍会对相同的文件和目录重复处理。

通过新的优化方案，系统会主动跳过不必要的搜索与扫描任务，进而降低磁盘输入输出（I/O）负载、减少中央处理器（CPU）周期占用，并削减后台索引任务的数量。最终，搜索过程中的整体资源消耗将显著下降，**内存占用量的降幅尤为可观**。

这些优化措施在正式发布前仍需进一步测试。不过按照微软常规的更新节奏，相关改进大概率会在 2026 年 1 月至 2 月期间，通过累积更新的方式推送给 Windows 11 稳定版用户。

需要说明的是，即便完成此次优化，文件资源管理器的搜索速度仍可能落后于 *Everything* 等第三方工具。对于需要频繁检索大量文件的用户而言，不妨考虑使用专业的第三方索引与搜索工具，以获得更快捷、更灵活的使用体验。
