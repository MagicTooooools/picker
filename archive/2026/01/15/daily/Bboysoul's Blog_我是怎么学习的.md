### 简介

经常有人问我是如何保持持续学习和获取新知识的，今天就来分享一下我的学习方法和信息获取渠道。在这个信息爆炸的时代，如何高效地筛选有价值的信息，避免信息过载，是每个技术人都需要面对的挑战。

### Newsletter 订阅

我订阅了几个高质量的 Newsletter，它们每天或每周会为我精选出最有价值的信息：

* **[TLDR](https://tldr.tech/)** - 科技行业的每日新闻摘要，快速了解科技圈的最新动态
* **[selfh.st](https://selfh.st/)** - 专注于自托管和开源项目的 Newsletter，适合喜欢折腾的人
* **[Kube Today](https://kube.today/)** - Kubernetes 相关的新闻和技术分享，对于云原生开发者来说是必读的

这些 Newsletter 的优点是信息已经经过筛选和整理，可以节省大量的信息过滤时间。

### GitHub Release 追踪

作为一名开发者，我会关注所有我感兴趣的 GitHub 项目的 Release 更新。GitHub 的 Watch 功能可以让你订阅项目的发布通知，这样就能第一时间了解到：

* 新功能的发布
* Bug 修复
* 重大更新

通过跟踪这些 Release，我能够保持对技术栈的最新理解，也能发现一些有趣的新特性。

我会定期整理自己的 Star 列表，按照分类管理感兴趣的项目。你可以在这里查看我关注的项目：

👉 [我的 GitHub Stars](https://github.com/bboysoulcn?tab=stars)

当然可以给我的Github点点关注

#### RSS 订阅

我维护了一个 RSS 订阅列表，可以在 [GitHub](https://github.com/bboysoulcn/rss) 上查看。通过 RSS，我可以订阅各种技术博客、新闻网站，然后将更新推送到 Telegram 频道。

#### 去重后的热门内容

为了避免信息重复，我对以下来源的内容进行了去重处理：

* **Hacker News** - 程序员社区的热门讨论和文章
* **GitHub Trending** - 每天最受欢迎的开源项目
* **V2EX 热榜** - 中文技术社区的热门话题

去重机制确保我不会看到昨天已经出现过的重复的内容，提高了信息消费的效率。

### 其他信息来源

除了上述主要渠道，我还会定期浏览：

* **YouTube 订阅** - 技术教程、开发者访谈、会议演讲等视频内容
* **Twitter** - 关注行业专家和技术领袖，获取第一手信息和观点。👉 [关注我的 Twitter](https://x.com/bboysoulcn)
* **Reddit** - 各种技术子版块，深度讨论和问题解答

### 关于信息过载

在信息爆炸的时代，信息过载是一个普遍问题。我的应对策略很简单：

**只看自己看得懂的**

这听起来很简单，但却是最重要的原则：

1. **不要强迫自己** - 如果一篇文章或一个技术点暂时看不懂，就先跳过，等有了相关基础再回来看，我一般会把想要reader later的文章放在linkwarden上<https://github.com/linkwarden/linkwarden>
2. **深度优于广度** - 与其浅尝辄止地看100篇文章，不如深入研究10篇真正有价值的内容
3. **实践为主** - 理论知识要结合实践，只有动手做过才能真正理解
4. **建立筛选机制** - 快速浏览标题和摘要，只深入阅读真正感兴趣的内容
